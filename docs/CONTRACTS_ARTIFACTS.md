# Xentient Artifact Contracts

> L2 Spec — Filesystem layout, DB schema, and Core↔Web interface for recorded session artifacts.

---

## Filesystem Layout

```
/var/xentient/artifacts/
  <spaceId>/
    <ulid>.wav      — raw PCM recording (S16LE, 16kHz)
    <ulid>.txt      — plain-text transcript
    <ulid>.md       — markdown session summary (optional, brain-generated)
```

- `<spaceId>` — matches the Space ID from SPACES.md (e.g. `living-room`)
- `<ulid>` — ULID generated by Core at session start; same ULID across all files of one session
- Core is the **only writer**. Web never writes artifacts.
- One session = one ULID = up to three files (not all are always produced)

---

## DB Schema

Table: `artifacts`

| Column          | Type       | Notes                                                   |
|-----------------|------------|---------------------------------------------------------|
| `id`            | TEXT PK    | ULID — matches the artifact filename prefix             |
| `spaceId`       | TEXT       | FK → spaces.id                                          |
| `triggerSource` | TEXT       | `"mic"` \| `"web_button"` \| `"scheduled"`             |
| `pipelineKind`  | TEXT       | `"basic"` \| `"hermes"` \| `"hermes+mem0"` \| `"openclaw"` \| `"archon"` |
| `startedAt`     | INTEGER    | epoch-millis uint32                                     |
| `endedAt`       | INTEGER    | epoch-millis uint32 (null while in-progress)            |
| `files`         | TEXT       | JSON array of relative paths: `["<spaceId>/<ulid>.wav", ...]` |
| `brainTier`     | TEXT       | Which brain tier handled this session                   |
| `status`        | TEXT       | `"recording"` \| `"complete"` \| `"error"`             |

## Core ↔ Web Interface

### Demo (Apr 24) — Shared SQLite

**Decision: shared SQLite file + shared filesystem path.**

- Core writes to `database/xentient.sqlite` (inside the Laravel app directory on Laragon)
- Web reads the same SQLite file via Eloquent — zero-config, no REST overhead for demo
- Both processes must be on the same machine (they are — Laragon)
- Web accesses artifact files directly from `/var/xentient/artifacts/` for audio playback via `<audio src>`

> **Known shortcut:** Direct filesystem + DB sharing violates process isolation (see WEB_CONTROL.md Auth Boundary). Acceptable for demo only. Platform track replaces this with Core REST API.

### Platform (post-demo) — Core REST API

Web uses Core REST endpoints; never touches the filesystem or DB directly.

| Endpoint | Method | Description |
|----------|--------|-------------|
| `GET /api/artifacts` | GET | Paginated list of sessions for a Space |
| `GET /api/artifacts/:id` | GET | Single artifact metadata |
| `GET /api/artifacts/:id/audio` | GET | Signed URL or streamed WAV (30-min expiry) |
| `GET /api/artifacts/:id/transcript` | GET | Plain-text transcript |

Web calls `GET /api/artifacts/:id/audio` and uses the returned signed URL as the `<audio src>` — it never constructs file paths manually.

## Sequence: Session Write (Core)

```
PIR trigger / web_button
  → Core Mode Manager sets mode = "record"
  → Core generates ULID
  → Core opens WAV file at /var/xentient/artifacts/<spaceId>/<ulid>.wav
  → Core inserts artifacts row (status = "recording")
  → Core streams PCM from INMP441
  → Core closes WAV, writes .txt transcript
  → Core updates artifacts row (status = "complete", endedAt = now)
```

## Sequence: Session Read (Web)

**Demo:**
```
Web /sessions page → Eloquent query on shared SQLite artifacts table
  → List cards with spaceId, startedAt, status, triggerSource
  → <audio src="/var/xentient/artifacts/<spaceId>/<ulid>.wav"> for playback
```

**Platform:**
```
Web /sessions page → GET /api/artifacts?spaceId=<id>
  → Click ▶ → GET /api/artifacts/:id/audio → signed URL
  → <audio src="<signed-url>">
```
