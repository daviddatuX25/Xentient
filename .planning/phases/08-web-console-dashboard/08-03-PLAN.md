# 08-03: Dashboard Overview Panel

## Goal

Build the main landing page of the Xentient Web Console showing system health at a glance: mode status, MQTT/camera/brain connectivity, live sensor gauges, active skills summary, and quick action buttons.

## Prerequisites

- **08-01 complete** — REST endpoints available
- **08-02 complete** — SSE events flowing

## Architecture Decision

Single-page HTML/JS dashboard served as static files from `public/`. No framework (React/Vue/Svelte). Vanilla JS + CSS using ES modules. No build step needed — `<script type="module" src="/js/main.js">` works natively in all modern browsers.

## Design System & Execution

**Design Reference:** `docs/xai-DESIGN.md`
**Execution Skill:** Use the `frontend-design` skill for all CSS implementation.

### Mode badge colors (override per-mode):
| Mode | Color |
|------|-------|
| sleep | `hsl(240, 60%, 55%)` (indigo) |
| listen | `hsl(160, 60%, 45%)` (emerald) |
| active | `hsl(40, 90%, 50%)` (amber) |
| record | `hsl(0, 70%, 55%)` (red) |

These mode colors are **exceptions** to the accent-blue-only rule — they are status indicators, not interactive elements.

### Elevation & Depth:
- Use gradient backgrounds and transparency for depth, NOT box-shadows
- Cards: `background: rgba(31, 34, 40, 0.7); backdrop-filter: blur(12px);`
- Use `border: 1px solid rgba(255,255,255,0.06)` for subtle card edges

### Font Loading (Expansion 3.3)

`universalSans` and `GeistMono` are not system fonts. Self-host WOFF2 files:

```html
<!-- In index.html <head> -->
<link rel="preload" href="/fonts/universalSans-Regular.woff2" as="font" type="font/woff2" crossorigin>
<link rel="preload" href="/fonts/GeistMono-Regular.woff2" as="font" type="font/woff2" crossorigin>
```

```css
/* In dashboard.css */
@font-face {
  font-family: 'universalSans';
  src: url('/fonts/universalSans-Regular.woff2') format('woff2');
  font-weight: 400;
  font-display: swap;
}
@font-face {
  font-family: 'GeistMono';
  src: url('/fonts/GeistMono-Regular.woff2') format('woff2');
  font-weight: 400;
  font-display: swap;
}
```

Fallback stack: `universalSans, -apple-system, system-ui, sans-serif` and `GeistMono, ui-monospace, monospace`. Fonts load async via `font-display: swap`.

### Step 3: Create main.js — state, routing, init

**File:** `harness/public/js/main.js` (NEW)

```javascript
import { DashboardAPI } from './api.js';
import { DashboardSSE } from './sse.js';
import { renderOverview } from './overview.js';
import { showToast } from './components.js';

// ─── State Management ───────────────────────────────
export const state = {
  mode: 'sleep',
  mqtt: false,
  camera: { online: false },
  brain: false,
  sensors: { temperature: null, humidity: null, pressure: null, motion: null, lastMotionAt: null },
  skills: [],
  activePack: null,
  connected: false,
  activeTab: 'overview',
};

// ─── Tab Routing (Expansion 3.6) ─────────────────────
// Hash-based routing enables deep linking, browser back/forward, bookmarkable state
const TABS = ['overview', 'skills', 'telemetry', 'mode'];

function setActiveTab(tab) {
  if (!TABS.includes(tab)) tab = 'overview';
  state.activeTab = tab;
  window.location.hash = tab;
  renderActivePanel();
  // Update nav active state
  document.querySelectorAll('#nav button').forEach(b => b.classList.toggle('active', b.dataset.tab === tab));
}

window.addEventListener('hashchange', () => {
  const hash = window.location.hash.slice(1);
  if (TABS.includes(hash)) setActiveTab(hash);
});

function renderActivePanel() {
  const content = document.getElementById('content');
  switch (state.activeTab) {
    case 'overview': renderOverview(content, state, api); break;
    // Skills, Telemetry, Mode panels added in 08-04, 08-05, 08-06
    default: content.innerHTML = '<p>Panel not yet implemented</p>';
  }
}

// ─── Init ────────────────────────────────────────────
const api = new DashboardAPI();
const sse = new DashboardSSE();

async function init() {
  // Show skeleton placeholders (Expansion 3.7)
  document.getElementById('content').innerHTML = renderOverviewSkeleton();

  // Fetch initial state from REST endpoints
  const [status, sensors, skills, packs, config] = await Promise.all([
    api.getStatus(),
    api.getSensors(),
    api.getSkills(),
    api.getPacks(),
    api.getConfig(),
  ]);
  Object.assign(state, status, { skills, activePack: packs.loaded, config });

  // Set initial tab from hash or default
  const hash = window.location.hash.slice(1);
  setActiveTab(TABS.includes(hash) ? hash : 'overview');

  // Connect SSE for real-time updates
  sse.connect();
}

init();
```

### Step 5: Gauge component — SVG with sensor ranges (Expansion 3.4)

Each sensor gauge has specific min/max ranges:

| Sensor | Min | Max | Unit |
|--------|-----|-----|------|
| Temperature | 0 | 50 | °C |
| Humidity | 0 | 100 | % |
| Pressure | 900 | 1100 | hPa |

```html
<div class="gauge" data-value="23.5" data-min="0" data-max="50" data-unit="°C">
  <svg viewBox="0 0 100 100">
    <circle class="gauge-bg" cx="50" cy="50" r="42" />
    <circle class="gauge-fill" cx="50" cy="50" r="42" />
  </svg>
  <span class="gauge-value">23.5</span>
  <span class="gauge-label">Temperature</span>
</div>
```

`stroke-dashoffset` calculation:
```javascript
const CIRCUMFERENCE = 2 * Math.PI * 42; // ≈ 263.9
const ratio = Math.max(0, Math.min(1, (value - min) / (max - min)));
const offset = CIRCUMFERENCE * (1 - ratio);
circle.style.strokeDashoffset = offset;
```

### Step 6: Quick action button feedback (Expansion 3.5)

When user clicks a quick action button, show immediate feedback before SSE confirmation:

1. Button enters **loading state**: disabled + spinner icon, text changes to "Switching..."
2. API responds (success or error)
3. If success: SSE confirms within ~200ms, button returns to normal
4. If error: show toast with error message, button returns to normal
5. **Timeout guard:** If no SSE confirmation within 5s, button returns to normal anyway (SSE might have missed it)

```javascript
async function handleQuickAction(btn, apiCall) {
  const original = btn.textContent;
  btn.disabled = true;
  btn.innerHTML = `<span class="spinner"></span> ${original}...`;

  try {
    await apiCall();
  } catch (err) {
    showToast(`Error: ${err.message}`, 'error');
  } finally {
    // SSE will update the state, but restore button after 2s max
    setTimeout(() => {
      btn.disabled = false;
      btn.textContent = original;
    }, 2000);
  }
}
```

## Edge Cases

- **Font files missing:** Fallback stack renders with system-ui. No layout shift if `font-display: swap` is used.
- **SSE reconnect during action:** If user clicks "Trigger Pipeline" and SSE reconnects mid-request, the button loading state times out after 2s and state is re-fetched.
- **Hash routing with unknown hash:** Falls back to `#overview`.
- **Module loading failure:** If any ES module fails to load, `import` will throw. Wrap `init()` in try/catch, show error toast.
