# 08-03: Dashboard Overview Panel

## Goal

Build the main landing page of the Xentient Web Console showing system health at a glance: mode status, MQTT/camera/brain connectivity, live sensor gauges, active skills summary, and quick action buttons.

## Prerequisites

- **08-01 complete** — REST endpoints available
- **08-02 complete** — SSE events flowing

## Architecture Decision

Single-page HTML/JS dashboard served as static files from `public/`. No framework (React/Vue/Svelte). Vanilla JS + CSS. Under 50KB total.

## Implementation

### Step 1: Create dashboard SPA shell

**File:** `harness/public/index.html` (NEW, replaces test.html as root)

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

<body>

    <header id="header"><!-- mode badge + connection status --></header>

  </div>
  <div id="reconnect-banner" class="banner hidden">Reconnecting...</div>
  <script src="/dashboard.js"></script>

```

**Key design decisions:**
- Tab navigation (Overview / Skills / Telemetry / Mode) — only one panel visible at a time
- Header shows current mode badge + connection indicators
- `reconnect-banner` appears when SSE disconnects
- Page title updates to show current mode: "Xentient — listen"

### Step 2: Create CSS file

**File:** `harness/public/dashboard.css` (NEW)

Design direction:
- Dark theme (GitHub Dark-inspired palette: `#0d1117` background, `#c9d1d9` text, `#21262d` cards)
- Glassmorphism cards: `backdrop-filter: blur(10px); background: rgba(33,38,45,0.7);`
- HSL-based color palette for mode badges:
  - sleep: `hsl(240, 60%, 55%)` (indigo)
  - listen: `hsl(160, 60%, 45%)` (emerald)
  - active: `hsl(40, 90%, 50%)` (amber)
  - record: `hsl(0, 70%, 55%)` (red)
- Responsive CSS Grid with breakpoints: mobile (<768px) single column, desktop multi-column
- Smooth CSS transitions for gauge animations (0.3s ease-out)

### Step 3: Create JS module

**File:** `harness/public/dashboard.js` (NEW)

Structure:

```javascript
// dashboard.js — Single-file SPA controller

// ─── SSE Client ─────────────────────────────────────
class DashboardSSE { /* ... reconnection, state recovery ... */ }

// ─── API Client ─────────────────────────────────────
class DashboardAPI {
  async getStatus() { return fetch('/api/status').then(r => r.json()); }
  async getSensors() { return fetch('/api/sensors').then(r => r.json()); }
  async getSkills() { return fetch('/api/skills').then(r => r.json()); }
  async setMode(mode) { return fetch('/api/mode', { method: 'POST', body: JSON.stringify({ mode }) }); }
  async triggerPipeline(source) { return fetch('/api/trigger', { method: 'POST', body: JSON.stringify({ source }) }); }
  async getPacks() { return fetch('/api/packs').then(r => r.json()); }
  async reloadPack(name) { return fetch(`/api/packs/${name}/reload`, { method: 'POST' }); }
}

// ─── Panel Renderers ────────────────────────────────
function renderOverview(state) { /* ... */ }
function renderSkillManager(state) { /* ... */ }
function renderTelemetry(state) { /* ... */ }
function renderModeControl(state) { /* ... */ }

// ─── State Management ───────────────────────────────
const state = {
  mode: 'sleep',
  mqtt: false,
  camera: { online: false },
  sensors: { temperature: null, humidity: null, pressure: null, motion: null, lastMotionAt: null },
  skills: [],
  activePack: null,
  connected: false,
};

// ─── Event Handlers ─────────────────────────────────
// SSE events update state, trigger re-render of active panel
// Tab clicks switch active panel
// Quick actions call API, state updates on SSE confirmation

// ─── Init ───────────────────────────────────────────
async function init() {
  // Fetch initial state from REST endpoints
  const [status, sensors, skills, packs] = await Promise.all([
    api.getStatus(),
    api.getSensors(),
    api.getSkills(),
    api.getPacks(),
  ]);
  Object.assign(state, status, { skills: skills, activePack: packs.loaded });
  renderOverview(state);
  // Connect SSE for real-time updates
  sse.connect();
}
```

### Step 4: Overview Panel Components

| └─ Reload pack button | `/api/packs/default/reload` | Instant |

### Step 7: SSE Reconnection + State Recovery

On SSE connect, fetch all REST endpoints for initial state. On reconnect:

```javascript
sse.on('connected', () => {
  // Re-fetch full state on reconnect
  api.getStatus().then(s => { state.mode = s.mode; state.mqtt = s.mqtt; });
  api.getSkills().then(s => { state.skills = s; });
  // ... etc
  renderActivePanel();
});
```
