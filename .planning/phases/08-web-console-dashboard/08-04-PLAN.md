## Components

### Skill List Table

- Sortable columns: ID, displayName, trigger type, priority, source, enabled, fireCount, lastFiredAt
- Row click → opens detail drawer (slide-in panel from right)
- Built-in skills (`source: 'builtin'`): delete button grayed out, tooltip "Cannot remove built-in skill"

### Sorting Implementation (Expansion 4.2)

Sortable columns with sort indicator arrows:

```javascript
class SkillManagerPanel {
  constructor() {
    this.sortColumn = 'id';
    this.sortAsc = true;
  }

  sortBy(column) {
    if (this.sortColumn === column) {
      this.sortAsc = !this.sortAsc;
    } else {
      this.sortColumn = column;
      this.sortAsc = true; // Default ascending for new column
    }
    this.render();
  }

  getSortedSkills() {
    const col = this.sortColumn;
    const dir = this.sortAsc ? 1 : -1;
    return [...this.state.skills].sort((a, b) => {
      const va = a[col] ?? '';
      const vb = b[col] ?? '';
      if (typeof va === 'number') return (va - vb) * dir;
      return String(va).localeCompare(String(vb)) * dir;
    });
  }
}
```

Column headers include sort indicator:
```html
<th onclick="panel.sortBy('id')">
  ID <span class="sort-icon">${sortColumn === 'id' ? (sortAsc ? '↑' : '↓') : '↕'}</span>
</th>
```

### Register Skill Form — Validation UX (Expansion 4.4)

When submitting a new skill, show validation feedback:

- **Inline errors** under each field with red border + error text
- Error message styling: `color: hsl(0, 70%, 55%); font-size: 12px; margin-top: 4px;`
- Red border on invalid fields: `border: 1px solid hsl(0, 70%, 55%)`
- Simple Mode validates: ID required, displayName required, trigger config complete
- Advanced Mode validates: JSON must parse, must match CoreSkill shape

```javascript
function validateSkillForm(formData) {
  const errors = {};
  if (!formData.id?.trim()) errors.id = 'Skill ID is required';
  if (!formData.displayName?.trim()) errors.displayName = 'Display name is required';
  if (!formData.trigger?.type) errors.trigger = 'Trigger type is required';
  // Per-trigger-type validation:
  if (formData.trigger?.type === 'event' && !formData.trigger?.eventName) {
    errors.trigger = 'Event name is required for event triggers';
  }
  if (formData.trigger?.type === 'interval' && !formData.trigger?.intervalMs) {
    errors.trigger = 'Interval is required for interval triggers';
  }
  return errors;
}
```

### Skill Removal Confirmation UX (Expansion 4.6)

Replace browser `confirm()` with inline confirmation — consistent with design system:

When delete is clicked, the delete button transforms into a red "Confirm Delete" button for 3 seconds:

```javascript
async requestRemoveSkill(skillId, btn) {
  if (btn.dataset.confirming === 'true') {
    // Second click — actually delete
    await this.api.deleteSkill(skillId);
    return;
  }
  // First click — show confirmation state
  btn.dataset.confirming = 'true';
  btn.textContent = 'Confirm Delete';
  btn.classList.add('btn-danger');
  // Reset after 3s if not confirmed
  setTimeout(() => {
    btn.dataset.confirming = '';
    btn.textContent = 'Delete';
    btn.classList.remove('btn-danger');
  }, 3000);
}
```

## Implementation

### File: `harness/public/js/skills.js` (NEW ES module)

```javascript
import { showToast } from './components.js';

export class SkillManagerPanel {
  constructor(api, sse, state) {
    this.api = api;
    this.sse = sse;
    this.state = state;
    this.sortColumn = 'id';
    this.sortAsc = true;
    this.detailSkill = null;
    this.drawerOpen = false;

    // SSE listeners
    this.sse.on('skill_registered', () => this.refreshList());
    this.sse.on('skill_removed', () => this.refreshList());
    this.sse.on('skill_updated', () => this.refreshList());
    this.sse.on('skill_fired', (d) => this.flashRow(d.skillId));
    this.sse.on('pack_loaded', () => this.refreshPack());
  }

  render(container) { /* ... builds table + pack + mappings HTML ... */ }
  refreshList() { this.api.getSkills().then(skills => { this.state.skills = skills; this.render(); }); }
  flashRow(skillId) { /* add .flash-accent class, remove after 500ms */ }

  async registerSkill(formData) {
    const errors = validateSkillForm(formData);
    if (Object.keys(errors).length > 0) { this.showFormErrors(errors); return; }
    // Check for collision — POST returns 409 if exists
    try {
      await this.api.registerSkill(formData);
    } catch (err) {
      if (err.status === 409) showToast(`Skill "${formData.id}" already exists. Use edit instead.`, 'error');
      else showToast(`Error: ${err.message}`, 'error');
    }
  }

  async toggleSkill(skillId, enabled) {
    await this.api.updateSkill(skillId, { enabled });
  }

  async requestRemoveSkill(skillId, btn) {
    if (btn.dataset.confirming === 'true') {
      try {
        await this.api.deleteSkill(skillId);
      } catch (err) {
        if (err.status === 403) showToast(err.data?.error || 'Cannot remove this skill', 'error');
        else showToast(`Error: ${err.message}`, 'error');
      }
      return;
    }
    btn.dataset.confirming = 'true';
    btn.textContent = 'Confirm';
    btn.classList.add('btn-danger');
    setTimeout(() => { btn.dataset.confirming = ''; btn.textContent = 'Delete'; btn.classList.remove('btn-danger'); }, 3000);
  }

  openDrawer(skillId) {
    this.detailSkill = this.state.skills.find(s => s.id === skillId);
    this.drawerOpen = true;
    // Update DOM
  }

  closeDrawer() {
    this.drawerOpen = false;
    this.detailSkill = null;
  }
}
```

### Trigger Form — Dynamic Fields

When trigger type changes, show/hide relevant fields. Types are loaded from `GET /api/config`:

| Trigger Type | Fields |
|-------------|--------|
| `event` | Event name (text) |
| `interval` | Interval ms (number) |
| `sensor` | Sensor key (dropdown: temperature/humidity/pressure/motion), operator (dropdown: >/</>=/<=/==/!=), value (number) |
| `mode` | From mode (dropdown from config.availableModes), To mode (dropdown from config.availableModes) |
| `cron` | Schedule (text, cron format) |
| `internal` | Event name (text) |
| `composite` | "Use Advanced Mode" button |

For `composite` trigger type, show a message: "Composite triggers require JSON configuration. Switch to Advanced Mode."

### Advanced Mode Toggle

Below the simple form, a toggle switches between:
- **Simple**: Form fields as described above
- **Advanced**: JSON textarea pre-populated with a CoreSkill template

The JSON textarea validates on submit — if invalid JSON, show inline error "Invalid JSON" below the textarea.
