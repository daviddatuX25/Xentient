# 08-04: Skill Manager Panel

## Goal

Build the Skill Manager panel providing full CRUD for the skill system, pack management, and event mapping management.

## Prerequisites

- **08-01 complete** — Skill, Pack, EventMapping REST endpoints
- **08-02 complete** — Skill/Pack lifecycle SSE events

## Components

### Skill List Table

- Sortable columns: ID, displayName, trigger type, priority, source, enabled, fireCount, lastFiredAt
- Row click → opens detail drawer (slide-in panel from right)
- Built-in skills (`source: 'builtin'`): delete button grayed out, tooltip "Cannot remove built-in skill"
- Pack skills (`source: 'pack'`): read-only, managed by Pack Loader
- Brain skills (`source: 'brain'`): full CRUD enabled
- Live fire indicator: when `skill_fired` SSE event arrives, flash the matching row green for 500ms

### Skill Detail Drawer

- Slide-in panel showing full skill config
- Sections: trigger config, actions list, escalation config, data collectors, modeFilter
- Read-only for builtin/pack skills; editable for brain skills

### Register Skill Form

- **Simple mode** (default): Form fields for common trigger types (event, interval, sensor)
  - ID (text input)
  - Display Name (text input)
  - Trigger Type (dropdown: event, interval, sensor, mode, cron, internal, composite)
  - Dynamic fields per trigger type (shown/hidden based on selection)
  - Actions (multi-add: type dropdown + config fields for set_mode, set_lcd, play_chime, mqtt_publish, log)
  - Priority (number input, default 50)
  - Cooldown (number input, ms)
  - Mode Filter (text input, optional)
- **Advanced mode** (toggle): JSON textarea for full CoreSkill config
- Submit calls `POST /api/skills`

### Enable/Disable Toggle

- Inline toggle switch per skill row
- Calls `PATCH /api/skills/:id` with `{ enabled: true/false }`
- SSE `skill_updated` confirms the change

### Pack Section

- Active pack card showing name and skill count
- "Switch Pack" dropdown (populated from `GET /api/packs`)
- "Reload" button (calls `POST /api/packs/:name/reload`)
- SSE `pack_loaded` / `pack_unloaded` updates pack name in real-time

### Event Mappings Section

- Table of current mappings: ID, source, eventName, protected flag
- Add custom mapping: source dropdown, eventName text input
- Remove custom mapping (protected ones show "Default" badge, no delete button)
- Calls `GET/POST/DELETE /api/event-mappings`

## Implementation

### File: `harness/public/dashboard.js` (ADD to existing)

Add a `SkillManagerPanel` class that renders into `#content` when the Skills tab is active:

```javascript
class SkillManagerPanel {
  constructor(api, sse, state) {
    this.api = api;
    this.sse = sse;
    this.state = state;
    this.sortColumn = 'id';
    this.sortAsc = true;
    this.detailSkill = null;

    // SSE listeners
    this.sse.on('skill_registered', (d) => this.refreshList());
    this.sse.on('skill_removed', (d) => this.refreshList());
    this.sse.on('skill_updated', (d) => this.refreshList());
    this.sse.on('skill_fired', (d) => this.flashRow(d.skillId));
    this.sse.on('pack_loaded', (d) => this.refreshPack());
  }

  render() { /* ... builds table HTML ... */ }
  refreshList() { this.api.getSkills().then(skills => { this.state.skills = skills; this.render(); }); }
  flashRow(skillId) { /* add .flash-green class, remove after 500ms */ }

  async registerSkill(formData) {
    const skill = this.buildSkillFromForm(formData);
    await this.api.registerSkill(skill);
    // SSE will fire skill_registered, triggering refreshList()
  }

  async toggleSkill(skillId, enabled) {
    await this.api.updateSkill(skillId, { enabled });
    // SSE will fire skill_updated
  }

  async removeSkill(skillId) {
    if (!confirm(`Remove skill "${skillId}"?`)) return;
    await this.api.deleteSkill(skillId);
    // SSE will fire skill_removed
  }
}
```

### Trigger Form — Dynamic Fields

When trigger type changes, show/hide relevant fields:

| Trigger Type | Fields |
|-------------|--------|
| `event` | Event name (text) |
| `interval` | Interval ms (number) |
| `sensor` | Sensor key (dropdown: temperature/humidity/pressure/motion), operator (dropdown: >/</>=/<=/==/!=), value (number) |
| `mode` | From mode (dropdown), To mode (dropdown) |
| `cron` | Schedule (text, cron format) |
| `internal` | Event name (text) |
| `composite` | "Use Advanced Mode" button |

For `composite` trigger type, show a message: "Composite triggers require JSON configuration. Switch to Advanced Mode."

### Advanced Mode Toggle

Below the simple form, a toggle switches between:
- **Simple**: Form fields as described above
- **Advanced**: JSON textarea pre-populated with a CoreSkill template

The JSON textarea validates on submit — if invalid JSON, show inline error "Invalid JSON".
