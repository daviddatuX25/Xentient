### SSE Event Tests (Expansion 8.1)

Test that REST mutations trigger SSE events. Use Node.js `EventSource` polyfill:

```typescript
import EventSource from 'eventsource'; // npm install --save-dev eventsource

describe('SSE Event Flow', () => {
  let es: EventSource;
  let events: SSEEvent[] = [];

  beforeEach(() => {
    es = new EventSource('http://localhost:3000/api/events');
    es.onmessage = (e) => events.push(JSON.parse(e.data));
  });

  afterEach(() => es.close());

  test('POST /api/skills triggers skill_registered SSE event', async () => {
    await fetch('/api/skills', { method: 'POST', body: JSON.stringify({ id: 'test-sse', ... }) });
    // Wait for SSE event
    await new Promise(r => setTimeout(r, 500));
    const registered = events.find(e => e.type === 'skill_registered' && e.skillId === 'test-sse');
    expect(registered).toBeDefined();
  });

  test('DELETE /api/skills/:id triggers skill_removed SSE event', async () => {
    // ... setup ...
    await fetch('/api/skills/test-sse', { method: 'DELETE' });
    await new Promise(r => setTimeout(r, 500));
    const removed = events.find(e => e.type === 'skill_removed' && e.skillId === 'test-sse');
    expect(removed).toBeDefined();
  });

  test('POST /api/mode triggers mode_change SSE event', async () => {
    await fetch('/api/mode', { method: 'POST', body: JSON.stringify({ mode: 'listen' }) });
    await new Promise(r => setTimeout(r, 500));
    const change = events.find(e => e.type === 'mode_change' && e.to === 'listen');
    expect(change).toBeDefined();
  });
});
```

## Performance Targets (Revised — Expansion 8.2, 8.3)

### Lighthouse Score

**Target: Lighthouse mobile score >70** (not >80). With custom fonts, canvas rendering, and SSE, >80 is unrealistic without significant optimization. Acceptable for v1.

Optimization measures:
- Lazy-load non-overview panels (Skills, Telemetry, Mode tabs load JS only when activated)
- Defer font loading with `font-display: swap` (already in 08-03)
- `<link rel="preconnect">` for SSE connection
- Canvas rendering deferred until tab is visible

### Payload Size

**Target: <50KB for HTML + CSS + JS** (fonts excluded). Font files are loaded async and cached by the browser after first load.

| Resource | Estimated Size |
|----------|---------------|
| index.html | ~2KB |
| dashboard.css | ~8KB |
| main.js | ~3KB |
| api.js | ~2KB |
| sse.js | ~3KB |
| components.js | ~5KB |
| overview.js | ~8KB |
| skills.js | ~10KB |
| telemetry.js | ~8KB |

| **Total (HTML+CSS+JS)** | **~55KB** |

Slightly over 50KB target. Acceptable for v1. Can be optimized later with minification.

## Edge Cases

- **EventSource polyfill in tests:** The `eventsource` npm package provides a Node.js implementation. Ensure it's installed as dev dependency.
- **Test server lifecycle:** Tests need a running ControlServer. Use `beforeAll`/`afterAll` to start/stop the server, or mock the HTTP layer.
- **Payload size measurement:** Measure with `wc -c` on built files, not source files. If ES modules add import overhead, measure the total transferred bytes.
- **Lazy panel loading:** If a panel module fails to load (network error), show an error toast and keep the current panel visible. Don't crash the whole dashboard.
