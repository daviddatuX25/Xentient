# 08-08: Integration Testing + Polish

## Goal

End-to-end verification that the dashboard works with live Core. Add polish: loading states, error toasts, keyboard navigation, favicon.

## Test Scenarios

| # | Scenario | Expected Result |
|---|----------|----------------|
| 1 | Start Core + open dashboard in browser | Dashboard loads with all 4 tabs, Overview shown |
| 2 | Click "listen" mode button | Mode badge changes to emerald "LISTEN" within 200ms via SSE |
| 3 | Register skill via dashboard form | Skill appears in skill list, SSE `skill_registered` confirms |
| 4 | Wait for BME280 sensor tick | Sensor gauges update with new values |
| 5 | Click "Trigger Pipeline" button | Skill fires, event feed shows fire entry |
| 6 | Load different pack via dropdown | Pack name updates, skill list reflects new pack |
| 7 | Disable a skill via toggle | Toggle grays out, skill stops firing |
| 8 | Close browser → reopen | State is correct (fetched from REST on load) |
| 9 | Kill Core process | "Reconnecting..." banner appears |
| 10 | Restart Core | Banner disappears, dashboard reconnects, state recovers |

## Polish Items

| Item | Implementation |
|------|---------------|
| Loading states | Skeleton screens (gray placeholder rectangles) while REST fetches complete. Not spinners. |
| Error toasts | `<div class="toast error">` slides in from bottom-right, auto-dismiss after 5s. Shown on API 4xx/5xx. |
| Keyboard navigation | Tab through skill list rows. Enter opens detail drawer. Space toggles enable/disable. Escape closes drawer. |
| favicon.ico | Xentient logo SVG → favicon (copy to `public/favicon.ico`) |
| Page title | Updates to "Xentient — {mode}" on mode change |
| Mobile responsive | CSS Grid collapses to single column on <768px. Tab bar becomes bottom nav on mobile. |
