## Hard Point H6: SpaceMode vs BehavioralMode UX

Two "mode" concepts exist:
- **SpaceMode** (hardware): sleep / listen / active / record — the state machine
- **BehavioralMode** (skill profile): default / student / teacher / etc. — the skill filter

**UX Decision:** Use distinct labels and visual sections:
- **"Hardware Mode"** — mode state machine diagram, 4-color badge system
- **"Skill Profile"** — dropdown selector, neutral gray styling, no color coding
- These are visually and semantically separated with distinct section headers and card backgrounds

### Step 2: Mode State Machine — SVG Diagram Generation

Generate SVG dynamically from `config.modeTransitions`:

```javascript
export function renderModeDiagram(container, currentMode, transitions, onModeClick) {
  // Layout: diamond-ish pattern
  const nodes = {
    sleep:   { x: 150, y: 50,  label: 'SLEEP',   color: 'hsl(240,60%,55%)' },
    listen:  { x: 150, y: 150, label: 'LISTEN',  color: 'hsl(160,60%,45%)' },
    active:  { x: 70,  y: 250, label: 'ACTIVE',  color: 'hsl(40,90%,50%)' },
    record:  { x: 230, y: 250, label: 'RECORD',  color: 'hsl(0,70%,55%)' },
  };

  const R = 30; // node radius
  let svg = `<svg viewBox="0 0 300 300" class="mode-diagram">`;

  // Draw edges (arrows)
  for (const [from, targets] of Object.entries(transitions)) {
    for (const to of targets) {
      const a = nodes[from], b = nodes[to];
      if (!a || !b) continue;
      const angle = Math.atan2(b.y - a.y, b.x - a.x);
      const x1 = a.x + R * Math.cos(angle);
      const y1 = a.y + R * Math.sin(angle);
      const x2 = b.x - R * Math.cos(angle);
      const y2 = b.y - R * Math.sin(angle);

      const isReachable = from === currentMode;
      const strokeColor = isReachable ? 'var(--text-secondary)' : 'rgba(125,129,135,0.2)';
      const strokeWidth = isReachable ? 2 : 1;

      svg += `<line x1="${x1}" y1="${y1}" x2="${x2}" y2="${y2}"
        stroke="${strokeColor}" stroke-width="${strokeWidth}"
        marker-end="url(#arrowhead)" />`;
    }
  }

  // Draw nodes
  for (const [id, node] of Object.entries(nodes)) {
    const isCurrent = id === currentMode;
    const isReachable = transitions[currentMode]?.includes(id);
    const opacity = isCurrent ? 1 : isReachable ? 0.9 : 0.3;
    const cursor = isReachable && !isCurrent ? 'pointer' : 'default';
    const glowFilter = isCurrent ? 'filter="url(#glow)"' : '';

    svg += `<circle cx="${node.x}" cy="${node.y}" r="${R}"
      fill="${node.color}" opacity="${opacity}"
      style="cursor:${cursor}" ${glowFilter}
      data-mode="${id}" class="mode-node ${isCurrent ? 'current' : ''} ${isReachable ? 'reachable' : ''}"
      onclick="handleModeClick('${id}')" />`;
    svg += `<text x="${node.x}" y="${node.y + 4}" text-anchor="middle"
      fill="var(--text-primary)" font-family="var(--font-mono)" font-size="11"
      style="pointer-events:none">${node.label}</text>`;
  }

  // SVG defs (arrowhead marker, glow filter)
  svg = `<defs>
    <marker id="arrowhead" markerWidth="8" markerHeight="6" refX="8" refY="3" orient="auto">
      <polygon points="0 0, 8 3, 0 6" fill="var(--text-secondary)" />
    </marker>
    <filter id="glow">
      <feGaussianBlur stdDeviation="4" result="blur" />
      <feMerge><feMergeNode in="blur" /><feMergeNode in="SourceGraphic" /></feMerge>
    </filter>
  </defs>` + svg + `</svg>`;

  container.innerHTML = svg;
}
```

### Step 3: Interactive SVG click handling (Expansion 6.3)

Each node has an `onclick` handler. Use event delegation for cleaner code:

```javascript
// In mode.js
function handleModeClick(targetMode) {
  if (targetMode === state.mode) return; // Already in this mode

  const validTransitions = state.config.modeTransitions[state.mode] || [];
  if (!validTransitions.includes(targetMode)) {
    // Invalid transition — show tooltip and brief red flash
    showTransitionError(state.mode, targetMode);
    return;
  }

  // Valid transition — call API
  attemptModeTransition(targetMode);
}

function showTransitionError(from, to) {
  showToast(`Cannot transition from ${from} to ${to} — must go through valid path`, 'error');
  // Brief red flash on the clicked SVG node
  const node = document.querySelector(`[data-mode="${to}"]`);
  if (node) {
    node.classList.add('invalid-flash');
    setTimeout(() => node.classList.remove('invalid-flash'), 500);
  }
}
```

```css
.invalid-flash { animation: flash-red 0.5s ease-out; }
@keyframes flash-red {
  0% { filter: brightness(2) hue-rotate(-60deg); }
  100% { filter: none; }
}
.mode-node.reachable:hover { filter: brightness(1.3); }
.mode-node:not(.reachable):not(.current) { cursor: not-allowed; }
```

### Step 4: Mode transition with loading state and failure handling (Expansion 6.7)

```javascript
async function attemptModeTransition(targetMode) {
  // Visual: immediately show the node as "transitioning"
  const node = document.querySelector(`[data-mode="${targetMode}"]`);
  if (node) node.classList.add('transitioning');

  try {
    const res = await fetch('/api/mode', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ mode: targetMode }),
    });

    if (!res.ok) {
      const err = await res.json();
      showToast(err.error || `Mode transition failed (${res.status})`, 'error');
      // Red flash on the target node
      if (node) {
        node.classList.add('invalid-flash');
        setTimeout(() => node.classList.remove('invalid-flash'), 500);
      }
    }
    // Success: SSE `mode_change` will update state and re-render diagram
  } catch (err) {
    showToast(`Network error: ${err.message}`, 'error');
  } finally {
    if (node) node.classList.remove('transitioning');
  }
}
```

### Step 5: Behavioral Mode Selector (Expansion 6.5)

The dropdown shows available behavioral modes. **Source:** `GET /api/spaces` returns the current space's behavioral mode. The list of available modes comes from `GET /api/config` or is derived from the `modeFilter` values in the skill list.

For v1, hardcode the list as a config constant (same as `PackSkillManifest` default modes):

```javascript
const BEHAVIORAL_MODES = ['default', 'student', 'teacher'];

// Render selector
function renderBehavioralModeSelector(container, currentMode) {
  container.innerHTML = `
    <section class="skill-profile-card">
      <h3 class="section-title">Skill Profile</h3>
      <select id="behavioral-mode" class="mode-select">
        ${BEHAVIORAL_MODES.map(m =>
          `<option value="${m}" ${m === currentMode ? 'selected' : ''}>${m.charAt(0).toUpperCase() + m.slice(1)}</option>`
        ).join('')}
      </select>
      <p class="hint">Changes which skills are active based on modeFilter</p>
    </section>
  `;

  document.getElementById('behavioral-mode').addEventListener('change', async (e) => {
    const mode = e.target.value;
    try {
      await fetch('/api/spaces/default/mode', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ mode }),
      });
    } catch (err) {
      showToast(`Error switching profile: ${err.message}`, 'error');
    }
  });
}
```

### Step 7: Hardware Status (Expansion 6.6)

Core doesn't track individual peripheral status. For v1, infer from data availability:

```javascript
function renderHardwareStatus(container, state) {
  const PERIPHERALS = [
    { id: 'bme280', label: 'BME280 (Climate)', key: 'temperature' },
    { id: 'pir', label: 'HC-SR501 (Motion)', key: 'motion' },
    { id: 'mic', label: 'INMP441 (Microphone)', key: 'audioLevel' },
    { id: 'lcd', label: 'LCD 16x2', key: 'lcdConnected' },
    { id: 'camera', label: 'ESP32-CAM', key: 'cameraOnline' },
  ];

  // Infer peripheral online: if last reading < 30s ago, it's "online"
  const lastSensorTime = state.sensors?.lastUpdate ?? 0;
  const espOnline = (Date.now() - lastSensorTime) < 30000;

  container.innerHTML = `
    <div class="hardware-status-card">
      <h3 class="section-title">Hardware</h3>
      <div class="status-row">
        <span class="status-indicator ${espOnline ? 'online' : 'offline'}"></span>
        <span>ESP32 NodeBase</span>
        <span class="font-mono text-secondary">${espOnline ? 'Online' : 'Offline'}</span>
      </div>
      ${PERIPHERALS.map(p => {
        const online = espOnline; // If ESP is online, assume all peripherals
        return `<div class="status-row peripheral">
          <span class="status-indicator ${online ? 'online' : 'offline'}"></span>
          <span>${p.label}</span>
        </div>`;
      }).join('')}
    </div>
  `;
}
```

## Edge Cases

- **API returns 400 for invalid mode:** Show toast "Invalid mode: {mode}" and don't change diagram.
- **SSE `mode_change` arrives during transition loading state:** Remove loading class, update diagram from SSE data.
- **Browser doesn't support `<script type="module">`:** Graceful degradation — show "Dashboard requires a modern browser" message. This applies to all frontend panels.
- **Config endpoint returns unexpected modes:** Diagram only renders nodes that exist in the hardcoded `nodes` map. Unknown modes in `modeTransitions` are ignored.
