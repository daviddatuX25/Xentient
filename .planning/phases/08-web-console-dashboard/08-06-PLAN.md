# 08-06: Mode & Space Control Panel

## Goal

Build the Mode & Space control panel with a visual state machine diagram, behavioral mode selector, space info, and hardware status indicators.

## Hard Point H6: SpaceMode vs BehavioralMode UX

Two "mode" concepts exist:
- **SpaceMode** (hardware): sleep / listen / active / record — the state machine
- **BehavioralMode** (skill profile): default / student / teacher / etc. — the skill filter

**UX Decision:** Use distinct labels and visual sections:
- "Hardware Mode" with mode state machine diagram
- "Skill Profile" with dropdown selector
- Different color schemes: Hardware Mode uses the 4-color badge system; Skill Profile uses neutral gray

## Implementation

### Mode State Machine — SVG Diagram

Hand-drawn SVG with 4 nodes and 7 edges (from `MODE_TRANSITIONS`):

```
         ┌───────┐
    ┌────│ sleep │◄───┐
    │    └───┬───┘    │
    │        ▼        │
    │   ┌─────────┐   │
    ├──►│ listen  │───┘
    │   └──┬──┬───┘
    │      │  │
    │      ▼  ▼
    │  ┌────────┐ ┌────────┐
    └─►│ active │ │ record │
       └────┬──┘ └──┬─────┘
            └──────┘
```

Implementation: SVG `<circle>` nodes + `<line>` edges. Current mode node has a glow animation (`box-shadow` + CSS animation). Invalid transitions from current mode are grayed out.

```javascript
const MODE_GRAPH = {
  nodes: [
    { id: 'sleep', x: 150, y: 50, color: 'hsl(240,60%,55%)' },
    { id: 'listen', x: 150, y: 150, color: 'hsl(160,60%,45%)' },
    { id: 'active', x: 80, y: 250, color: 'hsl(40,90%,50%)' },
    { id: 'record', x: 220, y: 250, color: 'hsl(0,70%,55%)' },
  ],
  edges: MODE_TRANSITIONS, // from contracts.ts
};
```

Click a valid target mode → `POST /api/mode` → SSE confirms. Invalid clicks show tooltip "Cannot transition from X to Y".

### Behavioral Mode Selector

```html
<section class="skill-profile">
  <h3>Skill Profile</h3>
  <select id="behavioral-mode">
    <option value="default">Default</option>
    <option value="student">Student</option>
    <option value="teacher">Teacher</option>
  </select>
  <p class="hint">Changes which skills are active based on modeFilter</p>
</section>
```

Calls `POST /api/spaces/:id/mode` to switch behavioral mode.

### Space Info Card

Shows current space configuration from `GET /api/spaces`:
- Space ID, NodeBase ID
- Active Pack name
- Current SpaceMode + BehavioralMode
- Integrations list
- Sensor list

### Hardware Status

Shows ESP32 connection state:
- Connection indicator (online/offline)
- Last heartbeat time
- Peripheral list with online/offline indicators (LCD, BME280, PIR, MIC, CAMERA)

**Note:** Core doesn't track ESP32 heartbeats currently. For v1, infer from last sensor message timestamp (if `sensorCache.lastMotionAt` or last BME280 reading is <30s ago, ESP32 is "online").

## Verification

1. State machine SVG renders with 4 nodes and correct transitions
2. Current mode node has glow effect
3. Clicking a valid transition (sleep → listen) fires `POST /api/mode` and SSE confirms
4. Clicking an invalid transition shows tooltip, no API call
5. "Skill Profile" section is visually distinct from "Hardware Mode"
6. Behavioral mode dropdown calls `POST /api/spaces/default/mode`
7. Space info card shows correct data from REST API
8. Hardware status shows last heartbeat time
