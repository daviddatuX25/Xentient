# 08-02: SSE Event Expansion

## Goal

Expand the SSE endpoint (`/api/events`) to push all events the dashboard needs for real-time updates. Add lifecycle events for skills, packs, and event mappings. Add throttled sensor updates and counter updates.

## Prerequisites

- **08-07 complete** — ControlServer must accept SpaceManager, EventBridge, PackLoader, SkillLog as deps

## Resolved Prerequisites (from Phase 7 post-implementation)

- **`broadcastSSE` visibility (Expansion 2.1):** `broadcastSSE` is `private`, but `broadcastSkillEvent()` is already `public` (line 253) and just calls `broadcastSSE()`. **Decision:** Rename `broadcastSkillEvent` → `broadcastSSE` and make it `public`. The current `broadcastSkillEvent` name is misleading since it broadcasts any SSE event, not just skill events. The old `private broadcastSSE` becomes the internal implementation detail.
- **EventBridge events (Expansion 2.3):** **CONFIRMED** — `mappingAdded` at line 217 and `mappingRemoved` at line 230 both exist with correct payloads. No additional work needed.

### Step 6: Add throttled SSE broadcaster

**File:** `harness/src/comms/ControlServer.ts`

Add throttling in ControlServer for sensor and counter events:

```typescript
private sensorThrottleTimer: NodeJS.Timeout | null = null;
private lastSensorBroadcast = 0;
private counterThrottleTimer: NodeJS.Timeout | null = null;
private lastCounterBroadcast = 0;
private static THROTTLE_MS = 1000; // 1 second

private broadcastThrottledSensor(data: object): void {
  const now = Date.now();
  if (now - this.lastSensorBroadcast >= ControlServer.THROTTLE_MS) {
    this.lastSensorBroadcast = now;
    this.broadcastSSE({ type: 'sensor_update', ...data });
  } else if (!this.sensorThrottleTimer) {
    this.sensorThrottleTimer = setTimeout(() => {
      this.sensorThrottleTimer = null;
      this.lastSensorBroadcast = Date.now();
      this.broadcastSSE({ type: 'sensor_update', ...data });
    }, ControlServer.THROTTLE_MS - (now - this.lastSensorBroadcast));
  }
}
```

### Step 7: Counter interval optimization (Expansion 2.2)

The 1s interval polls counters even when no skills have collectors, firing 86,400 times/day doing nothing. **Optimization:** Only start the interval when the first counter-using skill is registered, and clear it when the last one is removed.

```typescript
// In core.ts:
private counterInterval: NodeJS.Timeout | null = null;

spaceManager.on('skill_registered', (data) => {
  controlServer.broadcastSSE({ type: 'skill_registered', ...data });
  // Start counter polling if this skill has collectors
  const skill = spaceManager.listSkills().find(s => s.id === data.skillId);
  if (skill?.collect?.length && !this.counterInterval) {
    this.startCounterPolling();
  }
});

spaceManager.on('skill_removed', (data) => {
  controlServer.broadcastSSE({ type: 'skill_removed', ...data });
  // Stop counter polling if no skills have collectors
  const hasCollectors = spaceManager.listSkills().some(s => s.collect?.length);
  if (!hasCollectors && this.counterInterval) {
    clearInterval(this.counterInterval);
    this.counterInterval = null;
  }
});

private startCounterPolling(): void {
  this.counterInterval = setInterval(() => {
    const counters = spaceManager.getCounters();
    controlServer.broadcastSSE({ type: 'counter_update', counters });
  }, 1000);
}
```

10. Register first counter-using skill → counter polling interval starts

## Edge Cases

- **SSE client connects mid-stream:** Client receives `connected` event, then must fetch REST state for initial render. New lifecycle events only fire going forward.
- **Multiple SSE clients:** `broadcastSSE` iterates all clients; throttling is per-server (not per-client), so all clients get the same throttled rate.
- **PackLoader events with no listeners:** EventEmitter default is 10 listeners; if more are added, increase with `packLoader.setMaxListeners(20)`.
- **Counter interval lifecycle:** Interval starts on first `skill_registered` with `collect`, stops on last `skill_removed` with `collect`. Edge: what if a skill's collect is PATCHed in? SpaceManager should emit `skill_updated` with patch containing `collect` — core.ts checks if collect was added/removed.
- **`broadcastSSE` rename:** All existing callers in core.ts must be updated from `broadcastSkillEvent` to `broadcastSSE`. Missing update causes compile error (type-safe).
