# 08-01: Core REST API Expansion

## Goal

Add ~16 new REST endpoints to `ControlServer` that expose Phase 6-7 subsystems (SpaceManager, PackLoader, EventBridge, SkillLog) to any HTTP client. These endpoints mirror the existing MCP tools but are accessible via HTTP REST for dashboard consumption. Also adds a `GET /api/config` endpoint for frontend constants.

## Prerequisites

- **08-07 complete** ‚Äî MicroRouter + ControlServerDeps must be in place

## Resolved Prerequisites (from Phase 7 post-implementation)

- **`SpaceManager.listSkills()`** ‚Äî Already exists at `SpaceManager.ts:124`. Iterates all executors, returns flat `CoreSkill[]`. Optional `spaceId` parameter filters to one space. No additional work needed.

**Route ordering: `/api/skill-log` vs `/api/skills/:id` (Expansion 1.5)**

The original plan had `GET /api/skills/log` which conflicts with `GET /api/skills/:id` (the literal `log` would be captured as an `:id` parameter). Two options:

1. **Rename to `/api/skill-log`** ‚Äî separate path prefix, no collision possible
2. **Rely on registration order** ‚Äî register `/api/skills/log` before `/api/skills/:id`

**Decision:** Use `/api/skill-log` (Option 1). It eliminates the fragile registration-order dependency entirely. The MicroRouter resolves routes in registration order ‚Äî first match wins ‚Äî but relying on ordering is a footgun for future maintainers.

### Step 2: PATCH field allowlist (Expansion 1.2)

Not all skill fields are PATCHable. Dangerous fields that must NOT be changed via PATCH:

**Forbidden fields** (silently ignored or return 400):
- `id` ‚Äî identity, cannot be renamed
- `source` ‚Äî provenance, cannot change from `builtin` to `brain`
- `fireCount` ‚Äî observability counter, set by executor
- `escalationCount` ‚Äî observability counter, set by executor

**Patchable fields** (allowlist):
- `enabled` ‚Äî boolean toggle
- `displayName` ‚Äî string label
- `trigger` ‚Äî SkillTrigger object (full replacement)
- `actions` ‚Äî CoreAction[] (full replacement)
- `priority` ‚Äî number
- `cooldownMs` ‚Äî number
- `modeFilter` ‚Äî string | undefined
- `escalation` ‚Äî EscalationConfig | undefined
- `collect` ‚Äî DataCollector[]

```typescript
const PATCHABLE_FIELDS = new Set([
  'enabled', 'displayName', 'trigger', 'actions', 'priority',
  'cooldownMs', 'modeFilter', 'escalation', 'collect'
]);

private async handleUpdateSkill(req: IncomingMessage, res: ServerResponse, params: Record<string, string>): Promise<void> {
  const { id } = params;
  const body = await this.parseBody(req) as Record<string, unknown>;

  // Filter to allowlisted fields only
  const patch: Record<string, unknown> = {};
  for (const [key, value] of Object.entries(body)) {
    if (PATCHABLE_FIELDS.has(key)) patch[key] = value;
  }

  if (Object.keys(patch).length === 0) {
    this.sendJSON(res, 400, { error: 'No patchable fields provided' });
    return;
  }

  const skill = this.deps.spaceManager.listSkills().find(s => s.id === id);
  if (!skill) { this.sendJSON(res, 404, { error: `Skill '${id}' not found` }); return; }

  this.deps.spaceManager.updateSkill(id, patch);
  this.sendJSON(res, 200, { ok: true, patched: Object.keys(patch) });
}
```

### Step 3: POST skill ID collision ‚Üí 409 (Expansion 1.3)

If `POST /api/skills` is called with an ID that already exists, return **409 Conflict** (not silent overwrite). This differs from MCP `xentient_register_skill` which overwrites silently ‚Äî REST should be explicit.

```typescript
private async handleCreateSkill(req: IncomingMessage, res: ServerResponse): Promise<void> {
  const body = await this.parseBody(req) as Record<string, unknown>;
  const id = String(body.id ?? '');

  if (!id) { this.sendJSON(res, 400, { error: 'Skill ID is required' }); return; }

  // Check for existing skill
  const existing = this.deps.spaceManager.listSkills().find(s => s.id === id);
  if (existing) {
    this.sendJSON(res, 409, { error: `Skill '${id}' already exists (source: ${existing.source}). Use PATCH to update.` });
    return;
  }

  const skill: CoreSkill = { /* ... construct from body ... */ };
  this.deps.spaceManager.registerSkill(skill);
  this.sendJSON(res, 201, { ok: true, skill });
}
```

### Step 4: DELETE for pack skills ‚Üí 403 (Expansion 1.4)

The original plan only protected builtins (403). Pack skills should also be protected ‚Äî they are managed by PackLoader, not individual REST calls.

```typescript
private async handleDeleteSkill(req: IncomingMessage, res: ServerResponse, params: Record<string, string>): Promise<void> {
  const { id } = params;
  const skill = this.deps.spaceManager.listSkills().find(s => s.id === id);
  if (!skill) { this.sendJSON(res, 404, { error: `Skill '${id}' not found` }); return; }

  if (skill.source === 'builtin') {
    this.sendJSON(res, 403, { error: `Cannot remove builtin skill '${id}'` });
    return;
  }
  if (skill.source === 'pack') {
    this.sendJSON(res, 403, { error: `Cannot remove pack-managed skill '${id}'. Unload the pack instead.` });
    return;
  }

  this.deps.spaceManager.removeSkill(id);
  this.sendJSON(res, 200, { ok: true, removed: id });
}
```

### Step 6: Implement skill log query

The `SkillLog.query()` accepts `{ spaceId?, skillId?, since?, limit? }`. Expose as query params:

```typescript
private async handleGetSkillLog(req: IncomingMessage, res: ServerResponse, params: Record<string, string>): Promise<void> {
  const url = new URL(req.url!, `http://localhost`);
  const filter = {
    spaceId: url.searchParams.get('spaceId') ?? undefined,
    skillId: url.searchParams.get('skillId') ?? undefined,
    since: url.searchParams.get('since') ? Number(url.searchParams.get('since')) : undefined,
  };
  const limit = Number(url.searchParams.get('limit') ?? '100');
  const entries = this.deps.skillLog.query(filter, limit);
  this.sendJSON(res, 200, entries);
}
```

private async handleListPacks(_req: IncomingMessage, res: ServerResponse): Promise<void> {
  const available = this.deps.packLoader.listAvailablePacks();

private async handleLoadPack(req: IncomingMessage, res: ServerResponse, params: Record<string, string>): Promise<void> {
  const { name } = params;
  try {
    this.deps.packLoader.loadPack(name);
    this.sendJSON(res, 200, { ok: true, loaded: name });
  } catch (err) {
    this.sendJSON(res, 400, { error: String((err as Error).message) });
  }
}

## Edge Cases

- **Route ordering:** `/api/skill-log` is registered before `/api/skills/:id` to prevent `log` being captured as `:id`. This is now moot because the path prefixes differ entirely.
- **PATCH with no patchable fields:** Returns 400 with list of patchable fields.
- **PATCH with mixed fields:** Only patchable fields are applied; forbidden fields are silently stripped (not an error ‚Äî same behavior as JSON merge patch convention).
- **POST with missing required fields:** Returns 400 with field name.
- **GET /api/config caching:** Response is static (constants don't change at runtime). Frontend can cache indefinitely. No `Cache-Control` header needed for v1.
