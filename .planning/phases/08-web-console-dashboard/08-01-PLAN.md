# 08-01: Core REST API Expansion

## Goal

Add ~15 new REST endpoints to `ControlServer` that expose Phase 6-7 subsystems (SpaceManager, PackLoader, EventBridge, SkillLog) to any HTTP client. These endpoints mirror the existing MCP tools but are accessible via HTTP REST for dashboard consumption.

## Prerequisites

- **08-07 complete** — MicroRouter + ControlServerDeps must be in place

## Endpoints

### Skills (SpaceManager facade)

| Endpoint | Method | Handler | Description |
|----------|--------|---------|-------------|
| `/api/skills` | GET | `handleListSkills` | List all skills with state (fireCount, lastFiredAt, enabled, source) |
| `/api/skills/:id` | GET | `handleGetSkill` | Single skill detail by ID |
| `/api/skills` | POST | `handleCreateSkill` | Register a new skill (same schema as MCP `xentient_register_skill`) |
| `/api/skills/:id` | PATCH | `handleUpdateSkill` | Partial update (enable/disable, change trigger, etc.) |
| `/api/skills/:id` | DELETE | `handleDeleteSkill` | Remove skill (builtins protected) |
| `/api/skills/log` | GET | `handleGetSkillLog` | Query skill fire log (spaceId, skillId, since, limit) |

### Packs (PackLoader facade)

| Endpoint | Method | Handler | Description |
|----------|--------|---------|-------------|
| `/api/packs` | GET | `handleListPacks` | List available packs + which is loaded |
| `/api/packs/:name/load` | POST | `handleLoadPack` | Switch active pack |
| `/api/packs/:name/reload` | POST | `handleReloadPack` | Hot-reload current pack |

### Spaces (SpaceManager facade)

| Endpoint | Method | Handler | Description |
|----------|--------|---------|-------------|
| `/api/spaces` | GET | `handleListSpaces` | List spaces with mode + status |
| `/api/spaces/:id/mode` | POST | `handleSetSpaceMode` | Set behavioral mode for a space |

### Event Mappings (EventBridge facade)

| Endpoint | Method | Handler | Description |
|----------|--------|---------|-------------|
| `/api/event-mappings` | GET | `handleListEventMappings` | List all mappings (default + custom) |
| `/api/event-mappings` | POST | `handleAddEventMapping` | Register custom event mapping |
| `/api/event-mappings/:id` | DELETE | `handleRemoveEventMapping` | Remove custom mapping (protected ones rejected) |

### Sensor History (new)

| Endpoint | Method | Handler | Description |
|----------|--------|---------|-------------|
| `/api/sensors/history` | GET | `handleGetSensorHistory` | Get last 5 minutes of sensor readings (from SensorHistory ring buffer — see 08-05) |

## Implementation

### Step 1: Register new routes in ControlServer

Add to `registerRoutes()` method (established in 08-07):

```typescript
// Skills
this.router.add('GET',    '/api/skills',           this.handleListSkills)
this.router.add('GET',    '/api/skills/:id',       this.handleGetSkill)
this.router.add('POST',   '/api/skills',           this.handleCreateSkill)
this.router.add('PATCH', '/api/skills/:id',       this.handleUpdateSkill)
this.router.add('DELETE', '/api/skills/:id',       this.handleDeleteSkill)
this.router.add('GET',    '/api/skills/log',       this.handleGetSkillLog)
// Packs
this.router.add('GET',    '/api/packs',            this.handleListPacks)
this.router.add('POST',   '/api/packs/:name/load', this.handleLoadPack)
this.router.add('POST',   '/api/packs/:name/reload', this.handleReloadPack)
// Spaces
this.router.add('GET',    '/api/spaces',           this.handleListSpaces)
this.router.add('POST',   '/api/spaces/:id/mode',  this.handleSetSpaceMode)
// Event Mappings
this.router.add('GET',    '/api/event-mappings',   this.handleListEventMappings)
this.router.add('POST',   '/api/event-mappings',   this.handleAddEventMapping)
this.router.add('DELETE', '/api/event-mappings/:id', this.handleRemoveEventMapping)
// Sensor History (backed by SensorHistory — added in 08-05)
this.router.add('GET',    '/api/sensors/history',  this.handleGetSensorHistory)
```

**Route ordering matters:** `GET /api/skills/log` must be registered before `GET /api/skills/:id` to avoid `log` being captured as an `:id` parameter. The MicroRouter resolves routes in registration order — first match wins.

### Step 2: Implement handler methods

Each handler follows the same pattern:

```typescript
private async handleListSkills(req: IncomingMessage, res: ServerResponse, params: Record<string, string>): Promise<void> {
  const skills = this.deps.spaceManager.listSkills();
  this.sendJSON(res, 200, skills);
}
```

**Validation pattern** — Reuse Zod schemas from `contracts.ts`:

```typescript
private async handleCreateSkill(req: IncomingMessage, res: ServerResponse): Promise<void> {
  const body = await this.parseBody(req) as Record<string, unknown>;
  // Validate using existing Zod schemas
  const skill: CoreSkill = {
    id: String(body.id ?? ''),
    displayName: String(body.displayName ?? body.id ?? ''),
    enabled: true,
    spaceId: '*',
    source: 'brain',
    trigger: body.trigger as SkillTrigger,
    actions: (body.actions as CoreAction[]) ?? [],
    priority: Number(body.priority ?? 50),
    cooldownMs: Number(body.cooldownMs ?? 0),
    fireCount: 0,
    escalationCount: 0,
    collect: body.collect as DataCollector[] ?? [],
    escalation: body.escalation as EscalationConfig ?? undefined,
    modeFilter: body.modeFilter as string ?? undefined,
  };
  this.deps.spaceManager.registerSkill(skill);
  this.sendJSON(res, 201, { ok: true, skill });
}
```

**Error handling pattern:**

```typescript
private async handleDeleteSkill(req: IncomingMessage, res: ServerResponse, params: Record<string, string>): Promise<void> {
  const { id } = params;
  const skill = this.deps.spaceManager.listSkills().find(s => s.id === id);
  if (!skill) { this.sendJSON(res, 404, { error: `Skill '${id}' not found` }); return; }
  if (skill.source === 'builtin') { this.sendJSON(res, 403, { error: `Cannot remove builtin skill '${id}'` }); return; }
  this.deps.spaceManager.removeSkill(id);
  this.sendJSON(res, 200, { ok: true, removed: id });
}
```

### Step 3: Serialize EventMappings for API responses

The `EventBridge.listMappings()` returns `EventMapping[]` which includes functions (`filter`, `transform`). For the API, serialize to safe JSON:

```typescript
private serializeMapping(m: EventMapping): object {
  return {
    id: m.id,
    protected: m.protected ?? false,
    source: m.source,
    eventName: m.eventName,
    hasFilter: typeof m.filter === 'function',
    hasTransform: typeof m.transform === 'function',
  };
}
```

### Step 4: Implement skill log query

The `SkillLog.query()` accepts `{ spaceId?, skillId?, since?, limit? }`. Expose as query params:

```typescript
private async handleGetSkillLog(req: IncomingMessage, res: ServerResponse): Promise<void> {
  const url = new URL(req.url!, `http://localhost`);
  const filter = {
    spaceId: url.searchParams.get('spaceId') ?? undefined,
    skillId: url.searchParams.get('skillId') ?? undefined,
    since: url.searchParams.get('since') ? Number(url.searchParams.get('since')) : undefined,
  };
  const limit = Number(url.searchParams.get('limit') ?? '100');
  const entries = this.deps.skillLog.query(filter, limit);
  this.sendJSON(res, 200, entries);
}
```

### Step 5: Implement pack endpoints

```typescript
private async handleListPacks(_req: IncomingMessage, res: ServerResponse): Promise<void> {
  const available = this.deps.packLoader.listAvailablePacks();
  const loaded = this.deps.packLoader.getLoadedPack();
  this.sendJSON(res, 200, { available, loaded });
}

private async handleLoadPack(req: IncomingMessage, res: ServerResponse, params: Record<string, string>): Promise<void> {
  const { name } = params;
  try {
    this.deps.packLoader.loadPack(name);
    this.sendJSON(res, 200, { ok: true, loaded: name });
  } catch (err) {
    this.sendJSON(res, 400, { error: String((err as Error).message) });
  }
}
```
