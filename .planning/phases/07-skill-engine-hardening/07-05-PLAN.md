### Task 1: SkillExecutor gap fix tests

<read_first>
- `harness/tests/SkillExecutor.test.ts` (existing test patterns)
- `harness/src/engine/SkillExecutor.ts` (updated with G1-G5 fixes)
- `harness/src/shared/types.ts` (CoreSkill, SkillTrigger)
</read_first>

<action>
Create `harness/tests/SkillExecutor.gaps.test.ts` covering:

**G1 — Mode trigger evaluation:**
- Skill with `{ type: 'mode', from: 'sleep', to: 'listen' }` fires on `handleEvent('mode_transition', { from: 'sleep', to: 'listen' })`
- Wildcard mode trigger `{ type: 'mode', from: '*', to: 'listen' }` matches any → listen transition
- Wildcard mode trigger `{ type: 'mode', from: 'sleep', to: '*' }` matches any sleep→* transition
- Mode trigger does NOT fire for non-matching transitions (e.g., from: 'sleep', to: 'listen' doesn't match from: 'listen', to: 'active')

**G2 — Composite trigger evaluation:**
- Composite trigger with single sensor sub-trigger evaluates correctly
- Composite trigger with sensor + event sub-trigger evaluates only when both match
- Nested composite trigger works (depth 2)
- Composite trigger depth limit (depth > 5 returns false)

**G3 — modeFilter enforcement:**
- Skill with `modeFilter: 'student'` fires when `activeMode === 'student'`
- Skill with `modeFilter: 'student'` does NOT fire when `activeMode === 'family'`
- Skill without `modeFilter` (undefined) fires in all modes

**G4 — _idle-sleep removed:**
- `BUILTIN_SKILL_IDS` does not contain `'_idle-sleep'`
- `ALL_BUILTINS` has 3 entries (pir-wake, sensor-telemetry, determine-skill)
- ModeManager's idle timer still transitions listen→sleep after 60s

**G5 — DataCollector auto-collect + reset:**
- Skill with `collect: [{ type: 'counter', name: 'motionCount' }]` auto-increments counter on fire
- Skill with `collect: [{ type: 'counter', name: 'motionCount', resetAfterMs: 30000 }]` resets counter after 30s
- Counter reset timer restarts if skill fires again before reset
- Removing a skill clears its counter reset timers
- `SkillExecutor.stop()` clears all timers
</action>

<acceptance_criteria>
- All G1-G5 test cases pass
- Test coverage for `evaluateTrigger`, `matchesSpace`, `handleEvent` mode branch
- Counter reset timer tests use `vi.useFakeTimers()` for deterministic timing
