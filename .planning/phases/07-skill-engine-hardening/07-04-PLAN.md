# Plan 07-04: Skill Persistence

## Goal
Brain-registered skills persist across Core restarts. On boot, load persisted skills from `var/skills.json`. On skill registration/update/removal, write to the persistence file.

---

## Tasks

### Task 1: Implement SkillPersistence module

<read_first>
- `harness/src/engine/SkillExecutor.ts` (registerSkill, updateSkill, removeSkill)
- `harness/src/shared/types.ts` (CoreSkill interface)
</read_first>

<action>
Create `harness/src/engine/SkillPersistence.ts`:

```typescript
import fs from 'fs';
import path from 'path';
import pino from 'pino';
import type { CoreSkill } from '../shared/types';

const logger = pino({ name: 'skill-persistence' }, process.stderr);

export class SkillPersistence {
  private filePath: string;

  constructor(private dataDir: string) {
    this.filePath = path.join(dataDir, 'skills.json');
  }

  /** Load persisted skills from disk. Returns only brain-registered skills. */
  load(): CoreSkill[] {
    try {
      if (!fs.existsSync(this.filePath)) {
        logger.info('No persisted skills file found — starting fresh');
        return [];
      }
      const raw = fs.readFileSync(this.filePath, 'utf-8');
      const parsed = JSON.parse(raw);
      if (!Array.isArray(parsed)) {
        logger.warn('Persisted skills file is not an array — ignoring');
        return [];
      }
      // Only load brain-registered skills (pack and builtin are loaded from their own sources)
      const brainSkills = parsed.filter((s: CoreSkill) => s.source === 'brain');
      logger.info({ count: brainSkills.length }, 'Persisted skills loaded');
      return brainSkills;
    } catch (err) {
      logger.error({ err }, 'Failed to load persisted skills — starting fresh');
      return [];
    }
  }

  /** Save brain-registered skills to disk. */
  save(skills: CoreSkill[]): void {
    const brainSkills = skills.filter(s => s.source === 'brain');
    try {
      // Ensure data directory exists
      if (!fs.existsSync(this.dataDir)) {
        fs.mkdirSync(this.dataDir, { recursive: true });
      }
      fs.writeFileSync(this.filePath, JSON.stringify(brainSkills, null, 2), 'utf-8');
      logger.info({ count: brainSkills.length }, 'Persisted skills saved');
    } catch (err) {
      logger.error({ err }, 'Failed to persist skills');
    }
  }
}
```
</action>

<acceptance_criteria>
- `SkillPersistence` class reads/writes `var/skills.json`
- Only `source: 'brain'` skills are persisted (builtins and pack skills are loaded from their own sources)
- Missing file or malformed JSON doesn't crash — starts fresh
- Creates data directory if it doesn't exist
</acceptance_criteria>

### Task 2: Wire persistence into SkillExecutor lifecycle

<read_first>
- `harness/src/engine/SkillExecutor.ts`
- `harness/src/core.ts`
</read_first>

<action>
1. Add `SkillPersistence` as a dependency of `SkillExecutor` (optional, for testability).

2. In `SkillExecutor`, add persistence hooks:
   - After `registerSkill()` → persist
   - After `updateSkill()` → persist
   - After `removeSkill()` → persist

```typescript
// Add to SkillExecutorOptions:
export interface SkillExecutorOptions {
  // ... existing fields ...
  persistence?: SkillPersistence;  // optional — not set in tests
}
```

```typescript
// Add to SkillExecutor class:
private persist(): void {
  if (!this.opts.persistence) return;
  const allSkills = Array.from(this.skills.values());
  this.opts.persistence.save(allSkills);
}
```

Call `this.persist()` after each mutation:
- End of `registerSkill()`
- End of `updateSkill()`
- End of `removeSkill()` (after deletion)

3. On startup in `core.ts`, after creating the SpaceManager and SkillExecutor, load persisted skills:

```typescript
import { SkillPersistence } from './engine/SkillPersistence';

// In main():
const persistence = new SkillPersistence(path.join(process.cwd(), 'var'));
const persistedSkills = persistence.load();
for (const skill of persistedSkills) {
  spaceManager.registerSkill('default', skill);
}
```

4. Pass `persistence` to SkillExecutor via options when creating it in `SpaceManager`.
</action>

<acceptance_criteria>
- Brain-registered skills survive Core restart
- On boot, persisted skills are loaded and registered
- Persisted skills have `source: 'brain'`
- Builtin and pack skills are NOT duplicated in the persistence file
- If persistence file is corrupted, Core starts with builtins + pack skills only (no crash)
</acceptance_criteria>
