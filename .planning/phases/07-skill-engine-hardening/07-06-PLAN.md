## Task 4: Debounced persistence with 500ms batch window

**Items:** P2 (write amplification)
**Depends on:** Task 3

**type:** execute

<read_first>
- harness/src/engine/SkillPersistence.ts (after Task 3 changes)
- harness/src/engine/SkillExecutor.ts (lines 471-475 — persist() method, lines 60-69 — stop() method)
</read_first>

<action>
1. In SkillPersistence.ts, add two new methods after `save()`:

```typescript
private saveTimer: NodeJS.Timeout | null = null;

/** Debounced save — batches writes within 500ms window. */
debouncedSave(skills: CoreSkill[]): void {
  if (this.saveTimer) clearTimeout(this.saveTimer);
  this.saveTimer = setTimeout(() => {
    this.save(skills);
    this.saveTimer = null;
  }, 500);
}

/** Flush any pending debounced save immediately. */
flush(skills: CoreSkill[]): void {
  if (this.saveTimer) {
    clearTimeout(this.saveTimer);
    this.saveTimer = null;
  }
  this.save(skills);
}
```

2. In SkillExecutor.ts, change `persist()` (lines 471-475) to use debouncedSave:

```typescript
private persist(): void {
  if (!this.opts.persistence) return;
  const allSkills = Array.from(this.skills.values());
  this.opts.persistence.debouncedSave(allSkills);
}
```

3. In SkillExecutor.ts `stop()` method (around lines 60-69), add an immediate flush before the existing cleanup:

```typescript
// Flush any pending persistence writes before shutdown
if (this.opts.persistence) {
  const allSkills = Array.from(this.skills.values());
  this.opts.persistence.flush(allSkills);
}
```

The debounce reset pattern is correct: each `debouncedSave()` call replaces the pending timer with a new one closing over the latest `skills` array snapshot. Only the last snapshot within the 500ms window gets written.
</action>

## Task 7: Atomic validate-all-before-register in PackLoader

<action>
Restructure `loadPack()` to expand ALL skills before unloading+registering. Replace the method body (lines 22-41) with:

```typescript
loadPack(packName: string): void {
  const manifestPath = path.join(this.packsDir, packName, 'skills.json');
  if (!fs.existsSync(manifestPath)) {
    throw new Error(`Pack manifest not found: ${manifestPath}`);
  }
  const raw = fs.readFileSync(manifestPath, 'utf-8');
  const parsed = JSON.parse(raw);
  const manifest = PackSkillManifestSchema.parse(parsed);

  // Phase 1: Expand all skills — if any fails, nothing is loaded
  const expanded: CoreSkill[] = [];
  for (const ps of manifest.skills) {
    const skill = this.expandPackSkill(ps, manifest.pack.name);
    expanded.push(skill);
  }

  // Phase 2: All expansions succeeded — apply
  this.unloadCurrentPack();
  for (const skill of expanded) {
    this.registerFn(skill);
    this.loadedSkillIds.push(skill.id);
  }
  this.loadedPack = packName;
  logger.info({ pack: packName, skillCount: manifest.skills.length }, 'Pack loaded');
}
```

If `expandPackSkill()` throws (e.g., `_` prefix collision), the exception propagates before `unloadCurrentPack()` is called, so the old pack stays intact.
</action>
