### Task 4: Forward MQTT events to SkillExecutor

<read_first>
- `harness/src/core.ts` (MQTT event handler section)
- `harness/src/comms/MqttClient.ts` (event names emitted: 'vad', 'sensor', 'heartbeat', 'cameraFrame')
- `harness/src/engine/SpaceManager.ts` (handleEvent method)
</read_first>

<action>
In `core.ts`, after the existing MQTT event handlers, add forwarding to SpaceManager.

**Important:** MqttClient does NOT emit `'motion'` or `'vad'` events. Check `comms/MqttClient.ts` `handleMessage()` for the real event names. The actual events are:
- PIR motion → arrives via `mqtt.on('sensor', ...)` where `data.peripheralType === PERIPHERAL_IDS.PIR`
- VAD triggers → arrive via `mqtt.on('triggerPipeline', ...)` where `data.source === 'voice'`

```typescript
// Forward PIR motion to SkillExecutor event-triggered skills
mqtt.on('sensor', (data: unknown) => {
  const d = data as { peripheralType?: number; payload?: { motion?: boolean } };
  if (d.peripheralType === PERIPHERAL_IDS.PIR && d.payload?.motion) {
    spaceManager.handleEvent('*', 'motion_detected', { nodeId: mqtt.nodeId, timestamp: Date.now() });
  }
  // BME280 sensor update — forward for sensor-triggered skills
  if (d.peripheralType === PERIPHERAL_IDS.BME280) {
    spaceManager.handleEvent('*', 'sensor_update', { payload: d.payload, timestamp: Date.now() });
  }
});

// Forward VAD voice events to SkillExecutor
mqtt.on('triggerPipeline', (data: unknown) => {
  const d = data as { source?: string; stage?: string };
  if (d.source === 'voice') {
    const eventName = d.stage === 'start' ? 'voice_start' : 'voice_end';
    spaceManager.handleEvent('*', eventName, { timestamp: Date.now() });
  }
});

// Keep SpaceMode in sync when ModeManager transitions hardware state
modeManager.on('modeChange', ({ from, to }) => {
  spaceManager.updateSpaceMode('default', to);
  logger.info({ from, to }, 'SpaceMode synced with ModeManager');
});
```

**Note:** The existing `mqtt.on('sensor', ...)` in `events.ts` handles sensorCache updates. Add `spaceManager.handleEvent()` calls to the same handler — do NOT duplicate the handler, instead augment the existing one.
</action>
