# Xentient (X-1)

> **Modular AI Physicality.** A "show-don't-tell" interactive ecosystem for local, private, and modular hardware intelligence.

Xentient is a hardware architecture and software harness designed to bridge the gap between digital AI and physical environments. This project focuses on a modular "Node" approach where physical capabilities (Listening, Speaking, Seeing, Sensing) are snapped together to form a bespoke identity.

[Live Interactive Presentation 🌐](https://daviddatux25.github.io/Xentient/xentient_presentation.html)

---

## 📽️ Interactive Showcase

The primary entry point for understanding Xentient is the **Interactive Presentation Dashboard**. It replaces static documentation with hands-on simulations:

*   **⚡ Modular Node Builder:** A 3D-styled interface to drag and drop peripheral chips (Mic, Speaker, Camera, Sentinel) onto a Node Base. Features real-time power draw calculation and dynamic role assignment (e.g., "EAR NODE", "HYBRID NODE").
*   **📊 Real-time Flow Simulator:** Visualize the internal data pipeline (STT -> ID -> MEM -> LLM -> TTS) with contextual data markers and live event triggers.
*   **🏔️ Independence Stack:** An interactive 3D architecture model showing the 4 layers of Xentient—from physical hardware to the Cloud trust anchor.
*   **🏠 Space Contexts:** See how Xentient adapts to different environments like Home, Office, or Travel with real-world use case scenarios.

## 🚀 Getting Started

Simply open `xentient_presentation.html` in any modern web browser to explore the interactive showcase. 

### Key Interactions:
- **Click** on Architecture layers to lock-in detailed reading panes.
- **Drag & Drop** peripherals in the Builder to see the Node adapt.
- **Trigger** audio events to watch data packets flow through the Harness.

## 🌐 Vision
Xentient is built so the Cloud remains a **trust anchor, not a data trap**. By moving the majority of the "Harness" logic to local hardware, we ensure absolute privacy and resilience without sacrificing the power of modern LLMs.
